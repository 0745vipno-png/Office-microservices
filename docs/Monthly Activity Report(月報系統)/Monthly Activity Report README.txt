Monthly Activity Report

簡介

Monthly Activity Report 是一個本地端、唯讀、CLI 風格的工具，用於回顧指定月份內檔案系統的實際活動狀況。

本工具僅根據檔案系統的時間戳記推導結果，
不進行任何判斷、評分或建議行為。

工具定位

此工具在整個微服務生態系中的角色為：

長時間尺度的活動彙總觀察工具

它的目的在於回答：

指定月份內，大致有哪些檔案被建立或修改

檔案活動是否集中於特定日期

是否存在長期行為輪廓（而非單日或單週波動）

不適用於：

績效評估

工作量計算

自動化決策依據

設計原則

僅執行掃描與整理

不修改、不刪除任何檔案

不依賴其他工具是否曾執行

不假設使用者行為穩定或完整

所有輸出結果均為事實彙整，
後續判斷需由人自行進行。

使用方式
python monthly_activity_report.py <folder_path> <year> <month>

範例
python monthly_activity_report.py . 2026 1


搭配報告歸檔工具：

python monthly_activity_report.py . 2026 1 | python report_archiver.py monthly_activity

輸出內容說明
Summary

該月份內：

新增檔案數量

修改檔案數量

總活動筆數

Daily Breakdown

依日期列出：

當日新增的檔案

當日修改的檔案

僅列出實際發生過活動的日期

檔案過濾規則

預設僅列出常見辦公室文件格式（如 .docx, .xlsx, .pdf 等）

過濾規則可透過 daily_snapshot_config.json 設定

若設定檔不存在或格式錯誤，將自動回退至安全預設值

已知限制與風險說明
1. 檔案建立時間語意差異

Windows：ctime 通常代表檔案建立時間

Linux / macOS：ctime 可能代表 metadata 變更時間

因此：

本工具輸出的「新增檔案」為推導結果，
不保證在所有作業系統上語意完全一致。

2. 同一檔案的重複行為

若檔案在同一月份內「建立後又修改」
→ 僅列為「新增檔案」

此行為為刻意設計，避免在彙總層級產生混淆。

3. 非活動即不顯示

未發生新增或修改行為的日期，不會出現在報告中

這不代表該日「無工作」，僅代表檔案系統未留下可觀測事件

適用情境

月度回顧與自我檢視

長期工作行為觀察

作為人工檢討或報告撰寫的輔助資料

不適用情境（重要）

自動判斷工作成效

作為績效或考核依據

取代人為理解與解釋

若需要此類用途，不建議使用本工具。

最後說明

本工具僅負責提供：

「過去某段時間，檔案系統發生了什麼」

如何解讀、是否採取行動、以及行動後果，
均需由使用者自行判斷與承擔。
