Report Inventory

簡介

Report Inventory 是一個本地端、唯讀、CLI 風格的工具，用於盤點目前實際存在的報告檔案。

它僅掃描 reports 目錄結構，將「已存在的報告事實」整理並顯示出來，
不推論完整性、不進行判斷、不觸發任何後續行為。

工具定位

此工具在微服務生態系中的角色為：

報告狀態盤點工具（Report Inventory / Index）

它回答的唯一問題是：

「現在 reports 目錄裡，有哪些報告存在？」

設計原則

僅讀取檔案與目錄結構

不修改、不刪除任何檔案

不假設報告命名或時間語意一定正確

不推論「應該存在」或「是否缺漏」

所有輸出僅為現況描述。

使用方式
python report_inventory.py


預設掃描目錄：

./reports


或指定目錄：

python report_inventory.py <reports_dir>

輸出內容說明
報告類型（report_type）

每個子資料夾視為一種報告類型

例如：

reports/
  daily/
  weekly/
  monthly/

檔案列表

顯示該類型下實際存在的檔案名稱

若檔名符合 YYYY-MM-DD_HH-MM-SS.txt 格式，會嘗試解析並顯示時間

若無法解析，則僅顯示原始檔名

已知限制與邊界
1. 不判斷報告完整性

本工具不會判斷：

是否每天都有日報

是否每週或每月都有對應報告

是否存在缺漏或異常

這些判斷需由人自行進行。

2. 不假設報告產生時機

本工具不推論報告是即時產生或補跑

不嘗試區分「正常產生」與「重跑產生」

3. 檔名時間僅供參考

檔名中的時間僅為字串解析結果

不代表實際工作時間或事件發生時間

適用情境

快速盤點目前有哪些報告存在

檢視報告是否有重複產生

作為人工檢查或整理前的輔助工具

不適用情境（重要）

自動補齊缺失報告

作為排程或決策依據

取代人工檢視與理解

若需要上述用途，不建議使用本工具。

最後說明

Report Inventory 僅提供：

「目前檔案系統中，實際存在了哪些報告」

如何解讀這些結果、是否需要採取行動，
完全由使用者自行判斷與承擔。
